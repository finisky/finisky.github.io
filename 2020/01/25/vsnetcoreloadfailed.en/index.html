VS2017 .NET Core Projects Load Failed

Posted on 2020-01-25 Edited on 2022-05-02 In C# Views: Waline:

If you use Visual Studio 2017 and create an .NET Core projects the first time, the following errors might appear:

The current .NET SDK does not support targeting .NET Standard 2.0. Either target .NET Standard 1.6 or lower, or use a version of the .NET SDK that supports .NET Standard 2.0. XXXProject C:Files (x86)Visual Studio\2017.NET.Sdk.NET.TargetFrameworkInference.targets

Another related issue is .NET Core project load failed. Both errors point out that the corresponding .NET SDK is missing.

If you follow the instructions to install .NET Core SDK with a random version from here: https://aka.ms/dotnet-download . Most likely that the issue cannot be resolved.

It seems that there are many existing .NET Core 2.2.x SDKs. Why it still doesn't work?

Solution

Short Answer

  • Go to the following link to install SDK: # Download .NET SDKs for Visual Studio

  • Restart Visual Studio and rebuild the project

Long Answer

# The current .NET SDK does not support targeting .NET Standard 2.0 error in Visual Studio 2017 update 15.3

The above stackoverflow url has an answer which suggests to install .NET SDK from the following link (resolve the issue but don't know why): # Download .NET SDKs for Visual Studio

Compare the SDK download url with the offical .NET Core SDK site: # Download .NET Core 2.2, you will find that they are the same: 

https://dotnet.microsoft.com/download/dotnet-core/thank-you/sdk-2.2.110-windows-x64-installer

Looks magic, right? So I guess the reason is that Visual Studio 2017 is only compatible with a specific version of .NET Core SDK, regardless which version your project actually used. Fortunately the first site provides a compatible SDK version list for different VS versions, install from that site would resolve the problem.
